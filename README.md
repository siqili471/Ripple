# Ripple

Investigates how suppressing one LLM ability via contrastive decoding affects performance across all 18 ADeLe abilities.

## Contrastive Decoding Formula

```
logits_final = logits_neutral + α * (logits_neutral - logits_penalized)
```


## Project Structure

```
ripple/
├── config.py          # All config: model, paths, abilities, prompts
├── model.py           # Model loading, prompt building, logit inference
├── decoding.py        # Contrastive decoding + AUC computation
├── evaluate.py        # Evaluation loop with checkpoint/resume
├── visualize.py       # Heatmap generation
├── prepare_data.py    # Filter ADeLe battery → per-ability CSVs
└── run_experiment.py  # Main entry point
```

## Usage

**Step 1 — Prepare data** (only needed once):
```bash
python prepare_data.py
```

**Step 2 — Run experiment:**
```bash
python run_experiment.py --label "Qwen2.5-7B_Prompt1"
```

The experiment saves results incrementally after each discouraged ability. If the job crashes, re-running the same command will automatically skip completed abilities.

**Step 3 — Regenerate heatmaps only:**
```bash
python run_experiment.py --label "Qwen2.5-7B_Prompt1" --visualize-only
```

To compare multiple runs, point `visualize.py` at different CSVs:
```bash
python visualize.py --results results/results_7b.csv --label "Qwen2.5-7B_Prompt1"
python visualize.py --results results/results_14b.csv --label "Qwen2.5-14B_Prompt1"
```

## Results

Each run produces:
- `results/results.csv` — raw AUC scores (discouraged_ability × alpha × test_ability)
- `results/neutral_cache.pkl` — cached neutral logits (reused across runs)
- `figures/heatmap_<label>.pdf/.png` — 2×2 grid of heatmaps, one subplot per α
